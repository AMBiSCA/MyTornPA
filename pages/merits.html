Merit & Honor Tracker

Keep track of your hard-earned merits and honors in Torn City. Mark your achievements and see what's next on your list to conquer!

To make this Merit & Honor Tracker functional and appealing, consider these points:

  • Data Source for Merits/Honors:
    • You'll need a comprehensive list of all available merits and honors in Torn City. This is generally static data you can get from the Torn Wiki or community resources.
    • Organize them by category (e.g., Crimes, Stats, Company, General for merits; various categories for honors).
  • User Progress Storage:
    • Each user will need to save which merits/honors they have completed. This requires a database (like Firebase Firestore) to store a list of completed items linked to their `user.uid`.
  • User Interface (UI):
    • Display merits and honors clearly, perhaps in expandable sections by category.
    • Use checkboxes or a similar toggle for users to mark items as completed.
    • Show a progress bar or count (e.g., "You have completed X out of Y merits").
    • For each merit/honor, you could display its name, description, and the reward (e.g., +X% to Y stat for merits).
  • Core Functionality:
    • Load the full list of merits/honors.
    • When a user logs in, fetch their saved progress from your database and pre-select the completed items.
    • When a user checks/unchecks an item, save this change back to their profile in the database immediately or with a "Save Progress" button.
    • Allow filtering by category or searching for specific merits/honors.
  • User Experience (UX):
    • Make it visually clear which items are completed versus incomplete.
    • Provide a satisfying interaction when marking an item as complete.
    • Ensure the page is easy to navigate and update.
  • JavaScript Logic:
    • Dynamically render the list of merits/honors from your data source.
    • Handle user interactions (checking/unchecking).
    • Communicate with your database (Firestore) to save and load user progress.
    • Update progress displays (counts, progress bars).

A good start would be to get the full list of merits, display them with checkboxes, and then work on saving/loading the user's selections.
