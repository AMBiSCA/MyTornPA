Item Market Watcher

This tool will help you track item prices in Torn City, find good deals, and monitor market trends. Configure items you're interested in and let MyTornPA do the watching!

To make this Item Market Watcher functional and appealing, consider these steps:

  • Data Source: Determine how you'll get item market data. The Torn API provides access to item market listings (`torn/items/{item_id}/Torn API`). You'll need to handle API calls and potentially store/cache data frequently.
  • Item Selection:
    • Allow users to search for specific items by name or ID.
    • Provide a way for users to create a "watchlist" of items they want to monitor.
    • Perhaps categorize items (e.g., plushies, flowers, medical, weapons) for easier Browse.
  • Data Display:
    • Show current lowest listings, average prices, and perhaps highest buy prices.
    • Display price history charts for watched items (this would require storing historical data).
    • Clearly show item name, image, description, and market price.
  • Core Functionality:
    • Fetch and display real-time (or near real-time) market data for selected items.
    • Allow users to add/remove items from their watchlist.
    • Possibly implement price alert notifications (e.g., if an item drops below a certain price). This is more advanced.
  • User Interface (UI) and User Experience (UX):
    • Design a clean and intuitive interface for searching items and viewing data.
    • Ensure the page is responsive and works well on mobile.
    • Provide clear visual indicators for price changes (e.g., up/down arrows).
    • Make it easy for users to refresh data.
  • Backend/Storage (Potentially): For features like price history, user-specific watchlists, or price alerts, you would likely need a database (like Firebase Firestore) to store this information.

This is a complex tool to build, so start with core features like searching for an item and displaying its current market price, then expand from there!
