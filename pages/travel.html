Travel Helper

Plan your Torn City travels effectively! This tool will help you find profitable items in different countries, check stock levels, and calculate potential profits from your travel runs.

To build out this Travel Helper, you'll need to consider:

  • Data Sources:
    • Torn API: Use the `torn/items/{item_id}/` endpoint for item details. Foreign country item stock and prices are not directly available in a simple way via the standard user API for *all* items simultaneously.
    • Community Data/Third-Party APIs: Services like YATA often aggregate travel data. You might need to see if they offer an API or how you can leverage such community-sourced information (always respecting their terms).
    • Manual Updates/User Input: For some data, you might initially rely on curated lists or allow users to input prices they've seen.
  • User Interface (UI):
    • Allow users to select a destination country.
    • Display a list of notable items available in that country.
    • Show item images, descriptions, stock levels (if available), and prices in that country.
    • Show current Torn city market prices for comparison.
  • Core Functionality:
    • Calculate potential profit per item and per travel run (considering item capacity and travel time/cost).
    • Allow users to input their current travel capacity.
    • Filter items by category (e.g., plushies, flowers, drugs, special items).
  • Advanced Features (Future ideas):
    • Track user's current location and flight cooldowns (requires user API key with relevant permissions).
    • Notifications for when certain items are in stock.
    • "Best run" suggestions based on current data and user's capacity/time.
    • Integration with a personal inventory to see what they already have.
  • JavaScript Logic:
    • Fetch data from your chosen sources (API calls).
    • Perform calculations for profit, capacity, etc.
    • Dynamically update the HTML to display the information to the user.
    • Handle user selections (destination, items to track).

Start with a simple version, perhaps focusing on displaying items for one or two popular travel destinations and their Torn market prices. You can then expand data sources and features.
